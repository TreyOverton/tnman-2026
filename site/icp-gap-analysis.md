## 1. ICP vs. TOC — Why it matters for this conversation

| | **ICP (NIMS/ICS)** | **TOC (Army C2)** |
|---|---|---|
| Framework | NIMS / ICS | MDMP / FM 6-0 |
| Lead | Incident Commander (or Unified Command) | Commander, run by XO/S3 |
| Staff model | Command Staff (PIO, SO, LNO) + General Staff (Ops, Plans, Log, Fin/Admin, optional Intel/Investigations) | Coordinating, special, personal staff (S1–S9) |
| Scope | On-scene tactical incident management | Unit's operations across an AO |
| Key product | Incident Action Plan (IAP), 1 op period | OPORD / FRAGOs |
| Battle rhythm | Planning "P" / op-period cycle | MDMP and unit battle rhythm |
| Facility rules | One ICP per incident, named for the incident, outside hazard zone, not moved once established unless necessary | Sited per METT-TC; main CP / TAC / ALOC variants |

**Bottom line:** if the mission is DSCA / state active duty under a civil authority, expect to be evaluated against the ICP standard, not the TOC standard. We can run the ICP *out of* a TOC physically, but the org chart, products, and language the supported civil authority sees must be ICS.

## 2. Notional ICP Structure (what ICS-300 says "right" looks like)

### 2.1 Organizational components

- **Incident Commander (IC) / Unified Command** — overall management; sets objectives, approves IAP, approves resource requests.
- **Command Staff** (report directly to IC):
  - Public Information Officer (PIO)
  - Safety Officer (SO)
  - Liaison Officer (LNO)
- **General Staff** (Section Chiefs):
  - **Operations Section** — tactical execution; can expand into Branches → Divisions/Groups → Strike Teams/Task Forces/Single Resources.
  - **Planning Section** — situation, resources, documentation, demobilization; runs the planning P; produces the IAP. Units: Resources, Situation, Documentation, Demob (+ Technical Specialists as needed).
  - **Logistics Section** — Service Branch (Comms, Medical, Food) and Support Branch (Supply, Facilities, Ground Support).
  - **Finance/Administration Section** — Time, Procurement, Compensation/Claims, Cost. Activated only if the agency needs it.
  - **Intelligence/Investigations** — optional; can sit under Planning, as its own section, or under Ops depending on incident.

### 2.2 Facility / physical requirements

- **One ICP per incident**, named for the incident (e.g., "Operation X ICP").
- **Located outside the hazard zone** but close enough to maintain command.
- **Once established, do not move** unless absolutely necessary.
- **Incident Communications Center** normally co-located.
- May be co-located with the Incident Base; can be a building, trailer, vehicle, or tent.
- **Facilities Unit** (Logistics) sets it up and provides security/maintenance.

### 2.3 Required products and processes

- **Incident Action Plan (IAP)** for each operational period, built from ICS forms (201, 202, 203, 204, 205, 206, 207, 208, 215/215A, 230, etc.).
- **Operational planning cycle** ("planning P") — IC/UC objectives → tactics meeting → planning meeting → IAP prep → ops briefing → execute → assess.
- **Communications Plan (ICS 205).**
- **Resource check-in** (ICS 211) at ICP, Base, Staging, or via Resources Unit.
- **Operational period briefing** delivered using the IAP.
- **After-action report** on demobilization.

### 2.4 Minimum equipment / infrastructure (implied, not enumerated by ICS-300)

- Reliable comms (radio + commercial as backup), printer, plotter or large displays for maps and the org chart, ICS forms (paper and/or digital), status boards, sleep/rest rotation plan for 24-hr operations, security/access control, power, lighting, OPSEC for sensitive info.

## 4. Likely gap categories (for framing the CG's decision)

Most state / volunteer organizations with limited authorized strength will see gaps cluster in three places — worth pre-staging the discussion here:

1. **Depth, not breadth.** We can usually name *one* qualified person per box; doctrine assumes deputies and 24-hr coverage. Sustainment past ~12 hours is the first thing to break.
2. **Logistics Section.** Comms, Facilities, and Supply units are commonly under-resourced in non-federal organizations. Ground Support often does not exist.
3. **Finance/Admin and Intel/Investigations.** Frequently not resourced because the supported agency owns them — needs to be explicit in any agreement with the supported civil authority.
