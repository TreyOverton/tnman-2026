# ICP Task Organization Reference

> **Source file:** [ICP-Organization.pptx](/docs/source/ICP-Organization.pptx) — explanatory document provided by LTC Sheaf. This page is a Markdown rendering of the slide content for in-site reading. Use this as the template when drafting the 3 RGT [Task Organization](docs/taskorg-doc.md).

The slide describes the structure of an Incident Command Post (ICP) task organization and maps 3 RGT staff sections (S1–S6) to standard ICS positions.

## Org Chart


## Role Descriptions (from the slide)

### Incident Commander / TF Commander

Overall command authority for the incident / task force.

### Command Staff

| Role | Description |
|------|-------------|
| **Public Information Office (PAO)** | Responsible for communicating critical information to the public and media during emergencies, ensuring timely and accurate dissemination of information. |
| **Liaison Section (LNO)** | Be the POC for assisting cooperating agency representatives. Ensures effective operational support to facilitate collaboration and coordination among units and stakeholders. |
| **Safety Officer** | Responsible for a wide range of duties to ensure the safety and security of personnel, facilities, and resources. |

### Operations Section (S3)

Manages all tactical operations at an incident.

| Sub-element | Description |
|-------------|-------------|
| **Staging Area Manager** | Oversees all activities within a staging area, ensuring efficient organization and readiness of resources awaiting tactical assignments during emergency incidents. |
| **Air Operations Branch (4 / 3)** | Responsible for overseeing air operations during incidents, ensuring the safe and efficient use of aviation resources. |
| **SECFOR Task Force (3 / 3)** | 3 BN / 3 RGT as the SECFOR main effort at the HAAP perimeter. |

### Planning Section

Provides planning services for the incident.

| Sub-element | Description |
|-------------|-------------|
| **Situation Unit (S2)** | Oversees the collection, analysis, and dissemination of incident information, reporting to the Planning Section Chief and managing personnel to support incident planning and response. |
| **Resource Unit (S1)** | Tracks personnel and resources assigned to an incident, maintains resource status information, and supports planning activities throughout incident operations. |

### Support Section (S4)

Provides all incident support needs with the exception of logistics support to air operations.

| Sub-element | Description |
|-------------|-------------|
| **Communications Unit (S6)** | Signal, radio, data, and network support for the ICP. |
| **Medical Unit** | Medical support for the ICP and subordinate elements. |

## How to Use This Reference

1. Treat every role in the chart as a slot that must be filled (or explicitly marked **N/A** for this exercise).
2. For each slot, identify a named individual from the [Mission Roster](roster.md) and list their rank, name, and reach-back contact.
3. Where a slot cannot be filled from the current roster (e.g., PAO pending from DIV; dedicated Safety Officer; Staging Area Manager), flag it as TBD and add a line to [Planning Gaps](planning-gaps.md).
4. Map each slot to the standard 3 RGT staff section (S1–S6) so command-and-control language in the OPORD and on the radio stays consistent.
